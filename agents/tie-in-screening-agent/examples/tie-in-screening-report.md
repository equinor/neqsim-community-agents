# Example: Public Tie-In Screening Report Outline

This example uses synthetic public data for demonstration only.

## Input

- Concept: small subsea gas condensate tie-in to an existing host
- Flow rate: 0.5 to 1.5 million Sm3/day gas equivalent
- Pressure: 70 to 130 bara
- Temperature: 5 to 70 degC
- Fluid: methane-rich gas condensate with CO2 present
- Water: possible produced water during later life
- Host constraints: public placeholder only, no facility-specific data

## Expected Agent Actions

1. Review fluid data quality and missing composition details.
2. Flag hydrate screening considerations for low-temperature and water-present cases.
3. Identify separator screening questions for host inlet handling.
4. Suggest NeqSim workflows for fluid setup, phase behavior, hydrate screening, and simple separator checks.
5. List required follow-up studies.

## Example Output Summary

- Fluid data is insufficient for design because plus fraction, water content, and full contaminant data are not provided.
- Hydrate risk should be evaluated for low-temperature, water-present, cooldown, and restart cases.
- Separator screening should review gas and liquid handling range, slugging assumptions, and inlet pressure and temperature variation.
- Follow-up studies should include PVT validation, flow assurance, hydraulic analysis, process capacity checks, materials review, safety review, and project assurance.

## Human Review

A multidisciplinary engineering team must review the screening before concept selection or design decisions.
